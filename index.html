Featured Project Structure

Project Breakdown

SnapCal is positioned as my featured capstone project. The current release focuses on an Android MVP, while the long-term product direction expands toward a broader cross-platform service with future iOS and web support.

Project Summary

SnapCal is a mobile-first nutrition and activity tracking system that combines OCR-based food logging with imported health and workout data. The current implementation focuses on an Android MVP, while the system is designed to support future iOS and web expansion as part of a connected SaaS-style ecosystem.

Project Vision

To create a unified health tracking ecosystem where users can connect nutrition and physical activity data, reduce manual logging, and receive clear insights into calorie balance, macro intake, and progress over time.

Business / Project Requirements

  • Capture nutrition data from product labels using OCR.
  • Import activity data such as workouts, calories, and steps via Garmin / Health Connect.
  • Store nutrition and activity records locally for offline access.
  • Generate weekly reports with intake vs burn analysis and export options.
  • Provide manual correction and manual meal entry as fallback workflows.

Project Plan

  • Implement OCR capture and parsing for nutrition labels.
  • Integrate Garmin / Health Connect activity data.
  • Build local persistence and reporting features.
  • Refine UI/UX and test on modern Android versions.
  • Prepare documentation, status reporting, and final MVP presentation.

Requirements Analysis & Design

The core system revolves around entities such as User, Meal, Activity, and Report. The product flow supports authentication, OCR-based scanning, manual food entry, progress dashboards, and connected health data import. The architecture is intentionally separated into UI, processing, and storage layers to support future platform growth.
